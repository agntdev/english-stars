# fix-5ba1920e4f21bd37 — Race condition in saveQuizScore — concurrent completions lose entries

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 ENSTAR

`saveQuizScore` in `src/storage.ts:130-145` reads the current entries array, pushes a new entry, and writes it back. The `StorageAdapter` abstraction (read/write) has no atomic-append or CAS operation, so two quiz completions from the same user that arrive concurrently will see the same snapshot, and one write will overwrite the other, silently dropping a score entry. The same race exists in `incrementStat` (lines 97-102) but that is outside the E4 scope.

**Fix**: Either extend the `StorageAdapter` with an atomic `append`/`push` method (backed by PostgreSQL `INSERT` or Redis `RPUSH`), or use an alternative append-only storage path (e.g. a dedicated PostgreSQL table with `INSERT`).
